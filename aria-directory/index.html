Testing role=directory

Published: October 2, 2019

Tests were performed with the latest browser versions available at the time of publication, or as noted in the test results.

Note: there are no documented accessibility mappings to differentiate a directory from a standard list.

results

JAWS 2019 (aug)

When paired with Firefox and Chrome a "directory" will be discovered when navigating by lists, list items, and virtual cursor. It will be announced as a "list" with X items, making no mention of the "directory" role.

IE11 does not expose role=directory as a list, and JAWS does not mitigate this gap.

NVDA 2019.2.1

When paired with Firefox and Chrome a "directory" will be discovered when navigating by lists, list items, and virtual cursor. It will be announced as a "list" with X items, making no mention of the "directory" role.

IE11 does not expose role=directory as a list. For this pattern in particular, NVDA will ignore the role and instead respect the ul's mapping to expose this pattern as a list.

iOS 12.4.1 VoiceOver

The "directory" can be discovered if using the rotor to navigate by lists. When entering, VO will announce "list start" or "list end" depending on the entry point.

macOS 10.14.5 VoiceOver with Safari 12.1.1 and Chrome

The "directory" can be discovered if navigating by lists (control + option + command + X) or by standard VO navigation. When navigated to, VO will announce "list x items" or "end of list" depending on the entry point.

JAWS 2019 (aug)

When paired with Firefox and Chrome a "directory" will be discovered when navigating by lists, list items, and virtual cursor. It will be announced as a "list" with X items, making no mention of the "directory" role. When navigating to the "directory" (via list quick key or virtual cursor), the accessible name will be announced. The accessible name will not be announced when navigating by list item.

IE11 does not expose role=directory as a list, and JAWS does not mitigate this gap.

NVDA 2019.2.1

When paired with Firefox and Chrome a "directory" will be discovered when navigating by lists, list items, and virtual cursor. It will be announced as a "list" with X items, making no mention of the "directory" role. When navigating to the "directory" (via quick keys or virtual cursor), the accessible name will be announced.

IE11 does not expose role=directory as a list, and NVDA does not mitigate this gap.

iOS 12.4.1 VoiceOver

The "directory" can be discovered if using the rotor to navigate by lists. When entering, VO will announce "list start" or "list end" depending on the entry point. No accessible name will be announced by VoiceOver.

macOS 10.14.5 VoiceOver with Safari 12.1.1 and Chrome

The "directory" can be discovered if navigating by lists (control + option + command + X) or by standard VO navigation. When navigated to, VO will announce "list x items" or "end of list" depending on the entry point. The accessible name will be announced by VoiceOver when navigating to the "directory".
