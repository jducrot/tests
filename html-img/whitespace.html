Testing images with white space alt and aria-labels

Published: August 30, 2019

Expectation:

  • Images with alt=" ", alt="" or alt should receive no accessible name from the attribute.
  • Images with aria-label="" or aria-label=" " should receive no accessible name from the attribute.
  • Receiving no accessible name from alt or aria-label, all images should receive their accessible name from their title attribute.

Control: alt with empty string

results

Image should have name of "title 1".

  • Chrome: pass. No name. Ignored by screen readers.
  • Firefox: No name. Ignored by screen readers.
  • IE11:
  • macOS Safari: pass. Marked as "ignored".
  • macOS Chrome:
  • iOS Safari: pass. ignored
  • Android Chrome: pass. Ignored by TalkBack.
  • Android Firefox: pass. Ignored by TalkBack.

alt with single whitespace

results

Image should have name of "title 2".

  • Chrome: fail. Whitespace provides empty name. JAWS ignores, NVDA does not.
  • Firefox: fail. Whitespace provides empty name. JAWS ignores, NVDA does not.
  • IE11:
  • macOS Safari: pass. Marked as "ignored".
  • macOS Chrome:
  • iOS Safari: pass. ignored
  • Android Chrome: pass. Ignored by TalkBack.
  • Android Firefox: TalkBack exposes as nameless graphic.
